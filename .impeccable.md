## Design Context

### Users

Primary audience: potential clients, hiring managers, and startup founders evaluating Sami Bentaleb as a full-stack engineer. They arrive from LinkedIn, search, or referrals and need to quickly assess technical competence and creative range. The portfolio must communicate "this person is a pro" within seconds — visitors should leave feeling they've encountered someone cutting-edge and forward-thinking.

### Brand Personality

**Bold. Creative. Modern.**

Voice is confident and direct — no filler, no hedging. Copy should be concise and intentional, letting the work speak. Tone is professional but not corporate; technical but not dry. The portfolio should feel like walking into a well-designed studio: every element is there for a reason.

Emotional goal: visitors should feel a mix of admiration ("this is impressive") and trust ("this person knows what they're doing").

### Aesthetic Direction

**Japanese minimalism meets bold modernism.** Generous whitespace, restrained color palette, and precise typography — but with confident scale, strong contrast, and decisive layout choices. Nothing tentative or template-looking.

- **Theme:** Dark mode only. Deep, rich dark surfaces — not pure black.
- **Color:** Move away from default teal. Primary accent should be distinctive but restrained — used sparingly for maximum impact. Consider: electric blue, warm amber, or a sophisticated violet. The palette should feel curated, not default.
- **Typography:** Custom fonts required. Recommend geometric sans-serif for headings (Space Grotesk or Sora for their technical-yet-distinctive character) paired with Inter for body text. JetBrains Mono for code. Variable weights to create strong visual hierarchy.
- **Spacing:** Generous. Let elements breathe. Content widths (90%/75%/60% responsive) are already well-calibrated.
- **Motion:** Purposeful and subtle. CSS transitions for micro-interactions; consider activating the installed `motion` library for page transitions, scroll-reveals, and staggered list animations. No gratuitous animation — every movement should have a reason.
- **Imagery:** Project screenshots should be presented with care — consistent aspect ratios, subtle shadows or borders, hover states that reveal depth.

**References:** Designer portfolio sites that immediately communicate mastery. Japanese minimalism — the restraint of muji.com, the precision of a well-set Japanese book.

**Anti-references:** Generic developer portfolios with stock gradients, busy layouts, default Bootstrap/Tailwind templates, or anything that looks "made with a tutorial."

### Design Principles

1. **Intentionality over decoration.** Every element earns its place. If it doesn't serve the user or reinforce the brand, remove it.
2. **Confidence through restraint.** Bold doesn't mean loud. Use scale, contrast, and whitespace to create impact — not more colors, more effects, or more elements.
3. **Craft signals competence.** Pixel-perfect alignment, consistent spacing, thoughtful typography, and smooth interactions tell visitors "this person cares about quality" before they read a single word.
4. **Content hierarchy is king.** The eye should flow naturally: name → what I do → proof (projects) → depth (blog/about). Every page should have a clear focal point.
5. **Dark mode done right.** Not just "white text on black." Use layered surfaces (gray.900, gray.800, gray.750) to create depth. Ensure WCAG AA contrast on all text. Use the accent color to draw attention, not to fill space.

### Technical Standards

- **Accessibility:** WCAG 2.1 AA compliance. Minimum 4.5:1 contrast for body text, 3:1 for large text. Keyboard navigable. Screen reader compatible. Respect `prefers-reduced-motion`.
- **Performance:** Optimize images with Sharp/Next.js Image. Keep bundle lean. No unnecessary JS.
- **Responsive:** Mobile-first. Content must be excellent on all viewports — not just "not broken."

### Stack Reference

- **Framework:** Next.js 16 (App Router) + React 19 + TypeScript
- **Styling:** Chakra UI v3 (sole styling system, inline props)
- **Animation:** `motion` (Framer Motion) available but currently unused
- **Icons:** Lucide React + React Icons
- **Theme:** Dark mode only via next-themes
